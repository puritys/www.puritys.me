Chrome do not allow Cross-origin resource sharing for webfonts

Tags:
簡中版

CORS 全名 Cross Origin Resource sharing，定義了一些可以跨網域的 URI ，例如 Javascript, CSS, fonts，但是 Google 不知道那根筋不對，在新版的 Chrome 37.0.2062.94 開始，CSS 設定的 webfonts 就已經不能跨網域了，如果你的 webfonts 檔案是放在自已維護的伺服器，那麼你可以在 Apache conf 中加入 Allow-Origin 的設定，將你的 WebFonts 檔案設為允許跨網域。

Header set Access-Control-Allow-Origin: http://www.your.domain。

若是你有使用 「http://fonts.gstatic.com」的字型，就不用擔心這個問題， fonts.gstatic.com 的所有字型檔，都已經增加了允許所有網域的 header。

Access-Control-Allow-Origin: *
